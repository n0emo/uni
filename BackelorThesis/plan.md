# План выполнения ВКР

## Реферат

Проект представляет из себя среду выполнения для WASM компонентов и
предназначена для реализации высокопроизводительных и изолированных сетевых
сервисов, не требующих средств контейнеризации (docker, podman) и
виртуализации. WASM по своему дизайну является безопасной и изолированной
средой выполнения пользовательских программ. Для того, чтобы компоненты
могли выполнять полезную работу, необходимо обеспечить их библиотекой функций
для взаимодействия с внешними ресурсами (сетевые подключения, файловая система,
базы данных).

Программный продукт выполняет следующие функции:

- Загрузку, компиляцию и инстанциирование плагинов (WASM компонентов),
  экспортирующих функцию обработки входящего HTTP-запроса.
- Предоставление API для:
  - интерфейсов WASI;
  - получения доступа к конфигурации;
  - выполнения HTTP-запросов;
  - выполнения SQL-запросов;
  - выполнения Redis-запросов.
- Обработка HTTP запросов посредством инстанциирования плагина,
  соответствующего пути запроса.
- Автоматическая сборка стэка плагинов.
- Горячая перезагрузка на этапе разработки.
- Дашборд с техническими характеристиками сервера:
  - использование системных ресурсов (процессор, память, диск);
  - график количества обрабатываемых запросов в секунду;
  - график окна среднего времени обработки запросов;
  - логи сервера;
  - список загруженных плагинов;
  - диаграмма распределения обработанных запросов по плагинам;
